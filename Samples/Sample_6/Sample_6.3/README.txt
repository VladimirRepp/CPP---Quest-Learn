=== КОМПИЛЯЦИЯ И ЗАПУСК ===

Для компиляции (g++ или clang++):
g++ -std=c++26 -o game character.cpp heroes.cpp monsters.cpp main.cpp

Для запуска:
./game

=== СТРУКТУРА ПРОЕКТА ===

1. character.h/cpp     - Базовый класс и интерфейсы
2. heroes.h/cpp        - Классы героев (наследование)
3. monsters.h/cpp      - Классы монстров (полиморфизм)
4. main.cpp           - Демонстрация всех принципов ООП

=== ПРИНЦИПЫ ООП В ПРОЕКТЕ ===

1. ИНКАПСУЛЯЦИЯ:
   - Все поля классов защищены (private/protected)
   - Доступ только через публичные методы
   - Скрытие реализации (например, логика урона у Призрака)

2. НАСЛЕДОВАНИЕ:
   - Archer, Warrior, Mage наследуют Character
   - Ogre, Dragon, Ghost наследуют Character
   - Переиспользование кода из базового класса

3. ПОЛИМОРФИЗМ:
   - Виртуальные функции attack(), specialAbility()
   - Один метод работает для разных типов объектов
   - dynamic_cast для безопасного приведения типов

4. АБСТРАКЦИЯ:
   - Интерфейс IAttacker
   - Чисто виртуальные функции
   - Сокрытие сложности реализации
